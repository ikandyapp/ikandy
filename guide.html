📖 The Manual

User Guide

IKANDY paints your music. Press play in any music app and watch your screen come alive — colors, light and shapes that move with every beat. This guide walks you through everything: getting IKANDY installed, picking a music source, choosing presets and scenes, adding FX, and what to do when things misbehave.

01 — First Launch

Install &
First Run

IKANDY is a single-installer Windows desktop app. About 75 MB. No account required to launch it — you can start visualising system audio in under a minute.

Install

  1. Download IKANDY.Setup.exe from the home page or the GitHub releases page.
  2. Run the installer. It places IKANDY in your AppData folder and creates a Start Menu and desktop shortcut.
  3. The first launch may take a few extra seconds while the preset library indexes itself.
SmartScreen on first run Windows will show "Windows protected your PC" on the very first launch. This is because IKANDY is in beta and the Certum code-signing certificate is still in verification. Click More info → Run anyway. Once the certificate is issued, the warning disappears for everyone. The full source code is public at github.com/IKANDYapp/IKANDY.

What you'll see

A loading screen with the IKANDY mark fades in, then the main window opens with a default Butterchurn preset already running. The bottom of the screen shows the audio source selector. Pick a source to start hearing the visuals react to your music.

First-time tip The fastest path to seeing IKANDY work: start playing music in any app on your computer (Spotify, YouTube, anything), then in IKANDY pick Now Playing as the source. No setup, no auth — IKANDY listens to whatever Windows is currently playing and starts reacting immediately.

02 — The Layout

Where
Everything Lives

IKANDY is organised into four zones: a top bar, a left sidebar for lyrics, a centre stage for the visuals, and a right sidebar with all the controls.

Top Bar Logo · Status badge · Source selector · Hide toggle
Left Sidebar Lyrics Synced lyrics from LRCLIB. Switch between Classic and Physics modes.
Centre Stage The Show Your music, made visible. Full-bleed visuals at all times.
Right Sidebar Control Panel Master, Presets, Scenes, Vibe & FX. Each in its own collapsible accordion.

Top bar

The top bar is your nerve centre. The IKANDY logo sits left, with a status badge showing which source is connected. The source selector lets you switch between Spotify, foobar2000, VLC and Now Playing at any time. The hide button on the right collapses both sidebars for a clean fullscreen view.

Right sidebar — accordions

Click any accordion header to expand or collapse it. The ⊞ All button at the top expands or collapses every section at once. The four sections are:

  • Master — Reactivity and Smoothness sliders. These affect everything: presets, scenes, lyrics.
  • Presets — Auto-cycle interval, blend time, animation speed, visualizer opacity.
  • Scenes — Toggle and pick from the 5 custom audio-reactive scenes. Intensity slider.
  • Vibe & FX — Load a background image, then layer one of 12 GLSL shaders over it.
Bigger picture Hit Hide in the top bar to collapse both sidebars and let the visuals take the whole window. Click anywhere on the edge to bring the panels back.

03 — Audio Sources

Connect
Your Music

IKANDY can connect to four different sources. Each one offers a different feature set depending on what its underlying API exposes. Switching between them automatically pauses the previous one so you don’t get two streams playing at once.

Tier guide Full features — the richest experience. Playlist browse, search, every shuffle mode, transport, art, lyrics. Standard — transport, shuffle on/off, art, lyrics. No library browse. Basic — transport only, depending on what the source app supports. No browse, no shuffle, no art.
🎵 Spotify ● Full features · Premium recommended

The richest integration. Track info, album art, synced lyrics, playlist browser with search, drill-in to tracks, and full transport — play, pause, skip, shuffle modes (including Smart Shuffle detection), all the way down. IKANDY uses Bring Your Own Key auth: you create a free Spotify Developer app and paste the Client ID once. It’s saved encrypted on your machine via Windows DPAPI and never leaves your computer.

One-time setup

  1. In IKANDY, open the source selector and choose Spotify. The Spotify settings panel will show you the exact Redirect URI to register.
  2. Go to developer.spotify.com/dashboard, log in with your Spotify account, and click Create app.
  3. Give it any name (e.g. "My IKANDY") and any description. Under Redirect URIs, paste the exact URI shown in IKANDY.
  4. Check the Web API box. Save.
  5. Open the new app, go to Settings, copy the Client ID.
  6. Paste the Client ID into IKANDY's Spotify settings and click Connect. Spotify will open in your browser to authorise. After approval, you're connected.
Why BYOK Most apps embed a single shared Client ID, which means Spotify rate-limits everyone using that app together. With BYOK, your IKANDY uses your own quota — no sharing, no throttling, no API outages affecting you.
Premium required for transport Spotify's Web API only allows playback control (play, pause, skip, shuffle) for Premium subscribers. Free accounts will see track info, album art, and lyrics, but the playback buttons will be greyed out. It works with both Spotify desktop AND the web player — whichever Spotify Connect device is active will follow IKANDY’s commands.
🎧 foobar2000 ● Full features · Beefweb plugin

For the audiophile crowd. Connects to foobar2000 via the foo_beefweb component — a clean REST/JSON API that exposes the full library. Playlist browse with search, drill-in to tracks, every shuffle mode foobar supports (random, shuffle by track/album/folder, repeat playlist, repeat track), transport, album art, lyrics. The most feature-complete source IKANDY supports.

One-time setup

  1. Download and install the foo_beefweb component from github.com/hyperblast/beefweb/releases.
  2. Open foobar2000. Go to File → Preferences → Tools → Beefweb Remote Control.
  3. Tick Enable HTTP server. The default port is 8880.
  4. Optionally set a password under Authentication if you want to require one.
  5. Click Apply, then OK.
  6. In IKANDY, choose foobar2000, leave port at 8880 (or change to match), enter the password if set, and click Connect.
Why Beefweb Beefweb is the modern, actively-maintained way to control foobar2000 over HTTP. It supports album art, playlists, search, and every shuffle/repeat mode foobar offers natively.
📺 VLC ● Standard · Local HTTP

Connects to VLC’s built-in web interface for transport control over local files — play, pause, skip, shuffle on/off, album art, synced lyrics. No library browse: VLC’s playlist API isn’t expressive enough for IKANDY’s drill-in browser, so you control playback in IKANDY but pick what to play in VLC itself.

One-time setup

  1. Open VLC. Go to Tools → Preferences.
  2. At the bottom-left, switch Show settings from "Simple" to All.
  3. Navigate to Interface → Main interfaces and tick Web.
  4. Expand Main interfaces → Lua, set a Lua HTTP password (any password you'll remember), and save.
  5. Restart VLC. The web interface is now running on http://localhost:8080.
  6. In IKANDY, choose VLC, paste the password you set, and click Connect.
Custom port If port 8080 is taken on your system, you can change it in the same VLC preferences pane. Just match it in IKANDY's VLC settings.
Heads up If you mostly play music in a browser tab or in any other app besides VLC, Now Playing covers all of that without needing the VLC web interface set up. VLC’s dedicated source is most useful when you specifically use VLC as your music player.
🎙️ Now Playing ● Basic · Windows Media Controls · Zero setup

The catch-all. Reads whatever app is currently playing music on Windows — Spotify desktop, Spotify Web Player, YouTube Music, Apple Music, browser tabs (Chrome, Edge), Tidal, anything that publishes to Windows Media Controls. Track info shows up automatically; transport controls (play, pause, next, previous) work on whatever app is the current SMTC owner. No browse, no shuffle, no album art.

How to use it

  1. Start playing music in any app on your computer.
  2. In IKANDY, open the source selector and pick Now Playing.
  3. Within a second, the track info and transport controls light up. Hit play/pause/next/prev in IKANDY to control whatever app is playing.
What works depends on the app IKANDY dynamically shows or greys out individual transport buttons based on what the current source app supports. A YouTube single video may not expose Next/Previous, so those buttons grey out. A YouTube Mix or Spotify playlist will light them up. Hover any greyed-out button to see why it’s locked.
Firefox needs a flag Chrome and Edge bridge HTML5 Media Session to Windows Media Controls automatically. Firefox doesn’t by default — if you use Firefox, open about:config, search media.hardwaremediakeys.enabled, set it to true, and restart Firefox.
Windows only Now Playing relies on Windows Media Controls, which are part of Windows itself. The source isn’t available on macOS or Linux.

04 — Visualizer & Presets

350 MilkDrop
Presets

The core visualiser is built on Butterchurn 3.0 — a WebGL port of Winamp's legendary MilkDrop engine. The library ships with 350 presets, hand-picked from the MilkDrop community archive across the years.

Cycling presets

By default, IKANDY auto-cycles through presets at a fixed interval with a smooth blend between each one. Both knobs are in the Presets accordion in the right sidebar:

  • Auto-cycle interval — how long each preset stays on screen before moving to the next. Default is 30 seconds.
  • Blend time — how long the morph between presets takes. Longer blends feel cinematic; shorter blends feel kinetic.
  • Animation speed — global multiplier on preset motion. Boost it for a wilder ride, drop it for a chiller vibe.
  • Visualizer opacity — fade the preset back to let a background image or scene show through.

The preset browser

Click Presets in the top bar to open the full preset browser. All 350 are listed with thumbnail previews. Hover any thumbnail for a live preview, click to load it as the current preset. Thumbnails are pre-cached on first launch so the browser opens instantly after that.

Custom preset folder

If you've collected your own .milk files over the years, IKANDY can load them. In the Presets accordion in the right sidebar, point the Custom preset folder at any directory containing .milk files. They'll be merged into the browser alongside the bundled library, and the path persists across restarts.

Tip Some legacy MilkDrop presets use math that produces invalid values on modern hardware. IKANDY automatically skips broken presets after the first frame and remembers them — so you only see warnings once, and the cycle keeps moving smoothly.

05 — Custom Scenes

Audio-Reactive,
Built In-House

Five custom scenes, written from scratch for IKANDY. Unlike Butterchurn presets — which interpret a recipe of math — these are direct audio-reactive engines. Real-time FFT analysis on the input stream drives every pixel, frame by frame. The music literally makes the picture.

Turning scenes on

Open the Scenes accordion in the right sidebar. The ○ Off / ● On toggle at the top enables the scene layer. Below that, pick one of the five scenes. The Intensity slider controls how aggressively the scene reacts to the audio — from subtle ambient to maxed-out chaos.

Fire EQ
Frequency-reactive flames that climb with the spectrum. Low frequencies thicken the base, highs flick up sparks.
Spectrum
Classic ripple bars driven by live frequency analysis. Clean, readable, hypnotic.
Waveray
Radial beams that pulse and sweep with the waveform. Best with strong rhythmic music.
Aurora
Drifting curtains of light, mid-range driven. Soft, cinematic, hypnotic with vocal-led tracks.
Starfield
Beat-reactive cosmos. Stars sweep past while a central planet bounces with every kick — the harder the beat, the bigger the bloom.
Layering Scenes layer over Butterchurn presets. Drop the visualizer opacity in the Presets accordion to let the scene take centre stage, or keep it high to blend the two. Some of the most striking results come from running both at once.

07 — Lyrics

Two Modes,
One Source

IKANDY pulls synced lyrics from LRCLIB, a free, open community lyrics database. Lyrics fetch automatically based on the artist and title of the playing track — no manual entry, no per-track configuration.

Classic mode

One line at a time, centred on screen. Clean, karaoke-style. Perfect for recording, screen-sharing, or just reading along. The current line glows; previous and upcoming lines are dim.

Physics mode

Words explode onto the screen and float in zero-gravity. They collide, bounce off the edges, and drift in response to the audio. Totally chaotic, totally beautiful — best paired with a busy preset for maximum overload, or against a clean scene for legibility.

Switching modes Open the Lyrics accordion in the left sidebar. The mode toggle is at the top of the panel. You can flip between Classic and Physics mid-song.
No lyrics available LRCLIB is community-maintained, so very new releases or obscure tracks may not have synced lyrics yet. If a track has no LRC entry, IKANDY shows nothing instead of guessing.

08 — Master Controls & Themes

Tune the
Whole Thing

The Master accordion has two sliders that affect everything in IKANDY at once — every preset, every scene, every lyric animation. Get these dialled to your taste and the rest of the app feels right.

Reactivity

How aggressively the visuals respond to the audio. Low reactivity gives you a calm, ambient feel — visuals drift more than they react. High reactivity makes everything snap to the beat. Default sits in the middle and works for most music.

Smoothness

How much the audio analysis is averaged over time. High smoothness softens spikes and gives a more flowing feel; low smoothness makes the visuals chase every transient. For dub, ambient, or downtempo, crank smoothness up. For drum-and-bass or breakcore, drop it down.

Themes

IKANDY ships with 6 colour themes that re-skin the entire UI — sidebars, accordions, buttons, accents. The themes don't change the visuals themselves, just the chrome. Pick one that matches your music or your mood.

Set it once, forget it All settings persist across sessions. Reactivity, theme, source preferences, custom preset folder, last-used scene — all saved automatically. Next launch picks up where you left off.

09 — Tips & Workflow

Get the Most
Out of It

  • Selecting a Vibe & FX shader auto-disables Scenes. The image & shader path looks best on its own — IKANDY clears the scene layer automatically. Re-enable it manually if you want to layer.
  • Scenes layer over the Butterchurn visualizer. Drop the visualizer opacity to let the scene take centre stage, or keep it high to blend the two — some of the most striking results come from running both at once.
  • Drop visualizer opacity to spotlight scenes or backgrounds. The Butterchurn preset can fade to 30–50% to let a background image or scene breathe through.
  • Auto-cycle works while you're doing other things. Leave IKANDY running on a second monitor with auto-cycle on and forget about it — the show keeps changing.
  • Custom preset folder + your own .milk files. Decades of MilkDrop archive content still works. Drop a folder of .milk files into the custom preset path and they'll be merged into the browser.
  • Different sources, different lyrics. Spotify ships its own track metadata; foobar2000 and VLC use the file's tags; Now Playing reads whatever the source app reports to Windows. As long as title and artist are correct, lyrics fetch automatically. If lyrics aren’t loading, double-check the metadata on the playing source.
  • Volume control lives in the source app. IKANDY doesn’t adjust volume directly — use the source app’s own volume, or your Windows volume mixer. The volume slider in IKANDY is locked across all sources.
  • For recording or streaming. Classic lyric mode + a clean scene like Spectrum + a Holo or Plasma shader gives you a polished look that reads well on video.

10 — Troubleshooting

When Things
Get Weird

The most common issues people run into, and how to fix them. Click any item to expand.

Now Playing says "Nothing is playing"

This means Windows reports no app is currently registered as the SMTC (Windows Media Controls) owner. To fix, simply start playing music in any app (Spotify, YouTube tab, Apple Music, etc.) and the message will disappear within a second.

If music is playing and Now Playing still doesn’t see it, the source app probably isn’t publishing to SMTC:

  • Firefox — doesn’t bridge to SMTC by default. Open about:config, set media.hardwaremediakeys.enabled to true, restart Firefox.
  • Chrome / Edge — usually works automatically. If not, check chrome://flags for Hardware Media Key Handling and set it to Enabled.
  • Apple Music for Windows — can be inconsistent. Try pausing/resuming, or use the dedicated Spotify or foobar source instead if Apple Music is your main player.
Now Playing controls grey out / Next or Previous don't work

This is by design. IKANDY queries the active source app for which transport commands it actually exposes — some apps (single YouTube videos, some podcasts) don’t advertise Next or Previous, so IKANDY locks those buttons to be honest about what works.

Hover any greyed-out button to see the reason in the tooltip. If you want full transport, try a YouTube Mix or playlist URL, or use Spotify Web Player / Spotify desktop — those expose every transport command.

"Could not start audio source" / no audio reactivity

Even though IKANDY captures system audio (not your microphone), Windows treats them the same way for privacy purposes.

Fix: go to Settings → Privacy & security → Microphone and toggle Let desktop apps access your microphone on. Restart IKANDY.

If that doesn’t help, click 🎤 Reconnect System Audio in the Source panel to re-grab the loopback stream.

Spotify playback controls are greyed out

Spotify only allows external playback control for Premium subscribers — this is a Spotify API restriction, not an IKANDY one.

If you’re on a Free account you’ll still get track info, album art, and synced lyrics, but the play/pause/skip buttons will stay locked. Track info, lyrics, and visuals all still work normally.

Premium users: the dedicated Spotify source works whether you’re playing in Spotify desktop, Spotify Web Player, mobile, or a Spotify Connect speaker — whichever device is the active Connect target gets IKANDY’s commands.

Spotify won't connect / authentication fails

Most common cause: the Redirect URI in your Spotify Developer app doesn't exactly match the one IKANDY shows.

Open IKANDY's Spotify settings, copy the Redirect URI shown, then go to developer.spotify.com/dashboard, open your app, click Settings, and paste it into Redirect URIs. Save. Then re-attempt the Connect from IKANDY.

If that's not it, double-check the Client ID is pasted with no leading or trailing spaces, and that the Web API box is ticked in the Spotify app settings.

VLC won't connect

Make sure VLC's web interface is actually running. Open http://localhost:8080 in a browser — you should see the VLC web UI prompt for a password. If you don't, the Web interface isn't enabled.

Fix: VLC → Tools → Preferences → switch Show settings to All → Interface → Main interfaces → tick Web. Then expand Lua below it and confirm a Lua HTTP password is set. Restart VLC.

foobar2000 won't connect

Confirm Beefweb is installed and running. In foobar2000 → File → Preferences → Tools → Beefweb Remote Control, make sure Enable HTTP server is ticked.

Test by visiting http://localhost:8880 in a browser. If you see the Beefweb player, the plugin is healthy and IKANDY should connect with the same port.

If you set a Beefweb password, paste it into IKANDY's foobar2000 settings exactly as configured.

SmartScreen blocks the installer

"Windows protected your PC" — this happens because IKANDY is in beta and the Certum code-signing certificate is still in verification.

Fix: click More info, then Run anyway. The app is open source and the source is public on GitHub. Once the certificate is issued, future builds will be signed automatically and SmartScreen will stop warning everyone.

Visuals don't react to my music

Confirm the source is actually connected — the status badge in the top bar should show the source name (e.g. "Spotify") and a green dot.

Visuals react to system audio regardless of which source you’ve picked for track info. If music is playing but the visualizer is static, audio isn’t reaching IKANDY. Click 🎤 Reconnect System Audio in the Source panel, and check Windows microphone permissions (Settings → Privacy & security → Microphone → Let desktop apps access your microphone).

I have 5.1 or 7.1 surround sound and the visuals barely move

IKANDY captures system audio as a stereo (2-channel) loopback stream. If your output device is configured for 5.1 or 7.1 surround, Windows mixes most of the music into the rear/centre/sub channels, leaving the front L/R that IKANDY captures very quiet — so the visualizer barely reacts even though music is loud.

Fix: set your default output to Stereo (2 channel):

  1. Right-click the speaker icon in the system tray → Sounds.
  2. Open the Playback tab, right-click your active output device (speakers/headphones) → Configure Speakers.
  3. Choose Stereo, click Next through the wizard, then Finish.
  4. Back in the Playback tab, right-click the same device → Properties → Advanced, and pick a 2-channel format (e.g. 16 bit, 48000 Hz) from the Default Format dropdown.
  5. Click Apply, then in IKANDY hit 🎤 Reconnect System Audio.

Switch back to surround any time — the setting is just for IKANDY’s capture. Some users keep stereo as the default and only switch to surround for movies; others toggle as needed.

Lyrics aren't showing up

Lyrics come from LRCLIB, which is community-maintained. If the track is very new or obscure, there may not be a synced lyrics file yet.

You can confirm by searching for the track on lrclib.net directly. If LRCLIB has no entry, IKANDY can't display anything.

Also: lyrics need correct artist and title metadata. If your local files have wrong tags, lyrics will look up the wrong song.

Performance is choppy or stuttering

IKANDY is GPU-bound. If your GPU is busy with another task (game running, video encoding, another visualiser), frame rate will suffer.

Try: lower the Animation speed slider, drop Reactivity, or pick a simpler scene like Spectrum or Aurora instead of Starfield. Disable Vibe & FX shaders to remove a full-frame post-process pass.

On laptops, make sure IKANDY is using the discrete GPU and not the integrated one. Check via Windows Settings → System → Display → Graphics.

The app won't launch / crashes on startup

Try a clean reinstall: uninstall IKANDY via Settings → Apps, delete the AppData folder at %APPDATA%\IKANDY, and reinstall the latest build.

If it still crashes, open an issue on github.com/IKANDYapp/IKANDY/issues with your Windows version, GPU, and any error message that appears.
